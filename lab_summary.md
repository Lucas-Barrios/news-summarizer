This project was challenging because it required connecting several moving parts: external news fetching, two LLM providers, caching, scheduling, email delivery, analytics, and a web interface. I solved this by separating the system into focused modules, adding SQLite-backed caching to avoid duplicate LLM work, wrapping scheduled jobs with locks and logs, and testing the key behaviors with mocks and temporary databases. I learned how important clear boundaries are when building AI workflows that may later need to scale. Future improvements could include moving SQLite to Postgres or Supabase, adding stronger authentication for the web app, using a distributed task queue, and improving topic extraction with a dedicated NLP or LLM-based extractor.
